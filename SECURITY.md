# Security Policy

## Supported Versions

Only the latest version deployed to production is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email security concerns to: `bartek@smykla.com`
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Resolution target**: Within 30 days (depending on severity)

## Scope

This security policy covers:

- The klaudiu.sh redirect service
- The fly.io deployment configuration
- CI/CD workflows

## Out of Scope

- The klaudiush project (separate repository)
- Third-party dependencies (report to upstream maintainers)
